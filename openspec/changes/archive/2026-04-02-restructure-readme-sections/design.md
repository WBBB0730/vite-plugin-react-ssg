## Context

当前 README 在顶部简介后直接展示 Vite 配置示例，随后再进入安装与快速开始。对于首次接入的读者来说，信息顺序不够贴合“先安装、再接入、再看 API”的自然阅读路径；同时安装说明只覆盖 `pnpm`，API 区域也没有明确这些配置项属于 `defineReactSsgConfig(...)`。

本次调整只涉及文档信息架构，不涉及实现、类型或测试逻辑。中英文 README 需要保持相同的章节层级与内容映射，但中文 README 应使用中文标题，避免为了“结构一致”牺牲本地化阅读体验。

## Goals / Non-Goals

**Goals:**

- 统一中英文 README 的一级与二级标题结构及内容映射
- 把首次接入所需的安装与用法集中到 `Getting started`
- 在安装中同时提供 `pnpm` 与 `npm` 命令
- 在 API 区域明确配置项属于 `defineReactSsgConfig(...)`
- 保持现有信息基本完整，仅调整归类与顺序

**Non-Goals:**

- 不新增插件功能说明以外的营销性内容
- 不修改现有 API、行为描述和限制边界
- 不扩展新的示例文件或 playground 内容

## Decisions

1. 使用统一的信息架构，但让中文 README 采用中文标题。
原因：结构需要一致，但中文文档的可读性优先，标题应符合中文读者习惯。

2. 将 Vite 插件注册示例与 `react-ssg.config.ts` 示例统一放入“使用 / Usage”。
原因：这两类内容共同构成“完成接入”所需的最小步骤，放在同一段落最符合新手视角。

3. 在安装段落同时提供 `pnpm` 与 `npm` 两种安装命令。
原因：README 面向插件使用者，应覆盖更通用的安装方式。

4. 在 API 区域显式说明配置项属于 `defineReactSsgConfig(...)`。
原因：避免读者误解为这些选项是 `reactSsg()` 的参数。

5. 将输出行为与失败回退语义合并到 `Notes / 注意事项`，不再拆成独立章节。
原因：这两部分更像使用中的补充说明，而不是 API 本身。

6. 在 `Development / 开发` 段落中为每条命令补充简短作用说明。
原因：相比单独列命令，直接解释用途更方便贡献者快速理解开发流程。

## Risks / Trade-offs

- [风险] 章节重命名后，老读者需要适应新的标题名称。
  → 缓解：保留原有内容语义，只调整顺序与标题，减少理解成本。
- [风险] 中英文文档在重组过程中容易出现细节不一致。
  → 缓解：按同一映射关系同时调整两个 README，并在完成后逐段自查。
